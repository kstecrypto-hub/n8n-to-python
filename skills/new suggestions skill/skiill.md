SKILL: ADAPTIVE_ARCHITECTURE_INTELLIGENCE_ENGINE (AAIE)

PURPOSE:
Dynamically decide whether to reuse known architectural patterns or generate novel ones, and execute the appropriate strategy using structured search, mutation, and evaluation.

This skill replaces static reasoning with adaptive architecture intelligence.

---

STEP 0 — TASK CLASSIFICATION (MANDATORY FIRST STEP)

Classify the request along these axes:

- Novelty Requirement: {low, medium, high}
- Constraint Level: {tight, moderate, open-ended}
- Risk Tolerance: {low, medium, high}
- System Maturity: {early-stage, evolving, production}

---

STEP 1 — MODE SELECTION (CRITICAL CONTROL LOGIC)

Select ONE mode:

1. EXPLOIT MODE (use known patterns)
   Conditions:
   
   - Low novelty requirement
   - High reliability needed
   - Production system

2. EXPLORE MODE (full divergence)
   Conditions:
   
   - High novelty requirement
   - Open-ended problem
   - Architecture stagnation detected

3. HYBRID MODE (controlled innovation)
   Conditions:
   
   - Medium novelty
   - Existing system needs improvement
   - Balance between stability and innovation

You MUST justify the selected mode.

---

STEP 2 — FORBIDDEN PATTERNS (CONDITIONAL)

- In EXPLORE MODE → strict bans (≥3 patterns)
- In HYBRID MODE → soft constraints (discourage reuse)
- In EXPLOIT MODE → reuse allowed, but must be justified

---

STEP 3 — BASELINE MODELING

Define system state:

- Coordination model
- Agent topology
- Planning model
- Memory model
- Execution model
- Learning model

Also extract:

- bottlenecks
- rigidity points
- over-engineered components

---

STEP 4 — PARADIGM SPACE DEFINITION

Define applicable paradigms:

- Swarm
- Market
- Scheduler
- Constraint system
- Reflective/self-modifying

In EXPLOIT MODE:
→ select best-fit paradigm

In HYBRID:
→ select 2–3 to combine

In EXPLORE:
→ expand to maximum diversity

---

STEP 5 — ARCHITECTURE GENERATION STRATEGY

Depends on mode:

---

A. EXPLOIT MODE

- Refine existing architecture
- Optimize bottlenecks
- Improve coordination efficiency
- Introduce minimal structural changes

---

B. HYBRID MODE

- Apply controlled mutations:
  - modify 1–2 core assumptions
  - inject ONE new paradigm
- Maintain system stability
- Avoid full redesign

---

C. EXPLORE MODE

- Apply full mutation operators:
  - inversion
  - removal
  - substitution
  - cross-domain injection
  - constraint injection
  - dynamic mode shifting

Generate ≥5 radically different candidates.

---

STEP 6 — NOVELTY CONTROL

- EXPLOIT → minimize deviation
- HYBRID → enforce moderate divergence
- EXPLORE → enforce maximum divergence

Explicitly measure and justify difference level.

---

STEP 7 — EVALUATION

For each architecture:

- scalability
- failure modes
- complexity
- adaptability
- coordination efficiency

---

STEP 8 — DISAGREEMENT LOOP

Always run:

- Critic → identify weaknesses
- Challenger → propose alternative paradigm
- Defender → justify viability

---

STEP 9 — DYNAMIC COORDINATION MODEL (ADVANCED)

If HYBRID or EXPLORE:

Define:

- multiple coordination modes:
  {swarm, market, scheduler, constraint, reflective}
- conditions for switching
- system state triggers

---

STEP 10 — FINAL OUTPUT

Structure:

1. Task Classification
2. Mode Selection (with justification)
3. Baseline Architecture
4. Paradigm Selection
5. Generated Architectures
6. Novelty Justification
7. Evaluation
8. Disagreement Loop
9. (If applicable) Coordination Mode Switching Model
10. Final Recommendation

---

GLOBAL RULES:

- DO NOT overuse novelty when unnecessary
- DO NOT default to safe patterns without justification
- MATCH architecture complexity to problem requirements
- PRIORITIZE alignment with constraints over abstract creativity

---

CORE PRINCIPLE:

Intelligence is not constant creativity.

It is the ability to:

- reuse when appropriate
- innovate when necessary
- and know the difference
