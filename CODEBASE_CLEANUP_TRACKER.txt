  - `frontend/src/features/admin/sections/*`
  - `frontend/src/features/admin/AdminPage.tsx` as the shell-only page

Why each new module exists:
- `admin/*.py`: application-service ownership for admin workflows that are more than HTTP translation.
- `http_api/routes/admin/*.py`: route ownership by concern; HTTP translation only.
- `storage/agent_session_store.py`: session lifecycle, leasing, ownership, and token persistence only.
- `storage/agent_profile_store.py`: profile persistence, profile lookup, and profile token persistence only.
- `storage/agent_message_store.py`: transcript/message persistence only.
- `storage/memory_store.py`: typed session-memory persistence only.
- `storage/agent_trace_store.py`: query-run traces, replayable run detail, router/pattern state, and embedding cache persistence only.
- `storage/agent_feedback_store.py`: answer-review and feedback persistence only.
- `offline_pipeline/stages/source_prep.py`: source normalization/metrics stage only.
- `offline_pipeline/stages/extract_multimodal.py`: multimodal PDF extraction stage only.
- `offline_pipeline/stages/build_kg.py`: KG extraction/validation stage ownership only.
- `tests/test_architecture_boundaries.py`: import-boundary guardrails only.

What these modules must not own:
- admin services must not own raw SQL or route declarations.
- admin route modules must not own business logic, vector calls, or direct repository internals.
- storage modules must not import `http_api/*`, frontend code, or prompt modules.
- offline pipeline stages must not import `http_api/*` or chat-runtime prompt behavior.
- `PromptBuilder` and `AnswerVerifier` stay separate from storage/retrieval ownership.

- `src/bee_ingestion/api.py` remains a compatibility entrypoint that creates/exports the app and includes routers.
